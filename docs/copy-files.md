# worktree への追加ファイルのコピー（`copy`）

`git worktree add` は **追跡対象のファイルしか** 新しい worktree に持ってきません。そのため
`.gitignore` で除外している `.env` やローカルのシークレット・データディレクトリは、作成直後の
worktree には存在しません。`[copy]` セクションで「元リポジトリの作業ツリーから新しい worktree
へコピーするパス」を **リポジトリごと** に指定できます。

指定方法は **リポジトリごと** に 2 通りあります。値が **配列なら明示列挙**、**テーブルなら
オプション付き**（`.gitignore` 参照の自動モードを含む）です。

```toml
# ~/.config/worktree-integrator.toml

[copy]
# (A) 明示列挙: 配列でコピーするパスを並べる
"*" = [".env"]                       # すべてのリポジトリに適用
api = ["backend/.env", "frontend/.env"]

# (B) 自動モード: .gitignore を参照して無視ファイルを自動コピー
web = { gitignored = true, exclude = ["node_modules"] }
```

- **キー** は `repos_dir` 直下のリポジトリ名（厳密一致）。特別なキー `"*"` は **全リポジトリ** に
  適用され、各リポジトリ固有の指定とマージされます（`"*"` の分が先、重複は除去）。
- **値（配列形式）** はリポジトリルートからの相対パスの配列。**ファイルとディレクトリ**（再帰コピー）
  の両方を指定できます。
- **値（テーブル形式）** は次のキーを持てます:
  - `paths` … 明示パスの配列（配列形式と同じ。常にコピーされ、`exclude` の対象外）。
  - `gitignored` … `true` で **`.gitignore` を参照し、無視されている物を自動コピー** するモード。
  - `exclude` … 自動モードで除外する名前/パス（単一名 `node_modules` はどの階層でも一致、
    多段 `cache/tmp` は前方一致）。

## `.gitignore` 自動コピー（`gitignored = true`）

`.gitignore` で無視されているエントリ（`git ls-files -o -i` 相当）を自動的にコピーします。無視
ディレクトリは丸ごと再帰コピーされます。**既定の除外はありません** — `node_modules` や `.venv`、
`target` などの依存・ビルド成果物も、`exclude` に書かない限りそのままコピーされます。

- 除外したい物は `exclude` に追加します。`exclude` は **再帰的に適用** されるので、コピー対象の
  ディレクトリの **中にネストした** `node_modules` 等も取り除けます。
- `paths` に明示した物は常にコピーされ、`exclude` の影響を受けません。
- **注意**: 自動モードは対象が多くなりがちです。`node_modules` などを取り込みたくない場合は
  `exclude` に列挙して worktree の肥大化を避けてください。

## 共通の動作

- コピーは **worktree 作成直後**（`after_worktree` フックより前）に、選択された各リポジトリで実行
  されます。
- 元リポジトリに **存在しないパスは黙ってスキップ**（全リポジトリに同じ物がある必要はありません）。
- **追跡済みのファイルは書かない** でください。worktree が自動で持ってくるうえ、コピーすると
  作業ツリー側の内容で上書きされます。コピー対象は gitignore された未追跡ファイル向けです。
- **安全性**: 絶対パス・`..` を含むパス・`.` のみのパス・先頭が `.git` のパスは拒否します。さらに
  コピーは **コピー元・コピー先いずれの祖先シンボリックリンクも辿りません**。これにより、たとえ
  チェックアウトした worktree が悪意あるシンボリックリンクを含んでいても、worktree の外へ読み書き
  することはありません（シンボリックリンク自体はそのまま再作成されます）。
- コピーは **ベストエフォート**: 失敗しても worktree 作成自体は失敗扱いにならず、進捗に警告を
  出すだけです。コピーした項目は進捗に `コピー: <パス>` として表示されます。
- **既存 worktree への再呼び出し**（リポジトリ選択をスキップする場合）ではコピーは行われません
  （新規作成時のみ）。
