# ターミナル UI（`ui` サブコマンド）

サーバーログの閲覧・サーバー状態の監視・worktree の切り替えを 1 画面で行う
対話的なターミナル UI です。`server logs -f`（tail -f）がログを流すだけなのに対し、
`ui` は**表示を切り替えながら**読めます: サーバーごと／全サーバー混合の切り替え、
末尾追従の一時停止とスクロール、キーワードによる絞り込み、前世代ログ（`.prev`）の
参照をキー 1 つで行き来できます。

```sh
wt ui
```

![wt ui のデモ（ログの混合表示と色付け → 対象切り替え → フィルタ → ステータス → worktree の switch / stop）](assets/tui-demo.gif)

3 つのビューを `1` / `2` / `3`（または `Tab`）で切り替えます。終了は `q` です。

## ログビュー（`1`）

稼働中（または直近に稼働していた）サーバーのログを追跡します。表示は増分読み
（tail -f 相当の Go 実装）で、末尾 64KB から読み始めるため巨大なログでも即座に
開きます。

| キー | 動作 |
| --- | --- |
| `←` / `→`（`h` / `l`） | 表示対象の切り替え: **すべて**（全サーバーを到着順に混合、色付きタグつき）⇔ 各サーバー |
| `f` | 末尾追従（tail -f 相当）のオン／オフ。`j`/`k` などで上へスクロールすると自動で解除 |
| `/` | キーワードで絞り込み（部分一致・大文字小文字を無視。1 打鍵ごとに反映） |
| `p` | 前世代のログ（サーバー起動時にローテートされた `.prev`）との切り替え |
| `w` | 長い行の折り返しのオン／オフ |
| `j` / `k` / `d` / `u` / `g` / `G` | スクロール（1 行 / 半ページ / 先頭 / 末尾） |
| `Esc` | フィルタ解除 → worktree 絞り込み解除（の順） |

`error` / `warn` などの語を含む行は自動で色付けされます（ヒューリスティック）。

## ステータスビュー（`2`）

`server status` と同じテーブル（repo × server × worktree × 状態）を 2 秒ごとに
自動更新で表示します。`j` / `k` で行を選び、`Enter` でそのサーバーのログへ
ジャンプできます。

## worktree ビュー（`3`）

`list` と同じ worktree の一覧（別名・所属リポジトリ・稼働サーバーつき）です。
選択した worktree に対して:

| キー | 動作 |
| --- | --- |
| `Enter` / `s` | `server switch <name>` — その worktree へサーバーを切り替え |
| `r` | `server switch --restart <name>` — 既に稼働中でも再起動 |
| `x` | `server stop <name>` — その worktree のサーバーを停止 |
| `l` | その worktree に絞ったログビューへジャンプ |
| `R` | 一覧を再取得 |

切り替え・停止のライフサイクルイベント（起動・旧サーバー停止など）は画面下部の
イベント履歴に流れます。操作は 1 つずつ実行され、実行中の `q` は完了を待ってから
終了します。

## MCP・他プロセスとの兼ね合い

`ui` は MCP サーバーや別のシェルの CLI と**並行して動かす**前提で設計されています。

- **外部の変更に自動追従します。** 表示対象とログパスは定期的に再解決されるため、
  LLM エージェントが MCP の `server_switch` で worktree を切り替えると、ログ表示は
  数秒以内に新しい worktree のログへ自動で切り替わります（ステータス・worktree
  一覧も同様に追従します）。
- **操作は同じロックを通ります。** TUI からの switch / stop は CLI / MCP と同じ
  repo 操作ロックを通るため、エージェントの操作と衝突してもリポジトリ単位で
  直列化されます。TUI だけの特別な経路はありません。
- **MCP から TUI は起動できません。** `ui` は `server logs -f` と同じ CLI 専用の
  実行モードであり、MCP のツール語彙に存在しません（stdio を JSON-RPC が占有する
  MCP の下で全画面 UI が端末を奪う事故は型レベルで起こりません）。
- **設定はティックごとに再読み込みされます。** MCP サーバーと同じく、設定ファイルの
  編集は TUI の再起動なしで反映されます（編集途中で不正な間は直近の正常な設定で
  動き続けます）。

なお、TUI が端末を専有している間、ライフサイクルコマンド（`setup` / `on_switch`
など）の出力は表示されません（画面破壊を防ぐため破棄されます）。失敗時はエラーと
起動失敗のログ末尾がイベント・メッセージとして報告されます。詳細な出力が必要な
場合は CLI の `wt server switch` を使ってください。
