# ターミナル UI（`ui` サブコマンド）

worktree・サーバーの一覧と操作、選択サーバーのログ閲覧を 1 画面で行う対話的な
ターミナル UI です。lazygit 風の 2 ペイン構成で、左に worktree → サーバーのツリー、
右に選択中サーバーのログを表示します。worktree の切り替え・停止に加え、worktree の
作成・削除・別名・doctor もこの画面から実行できます。

```sh
wt ui
```

引数を一切与えずに `wt` だけを実行しても、同じ画面が開きます（lazygit などと同様、
素の起動は最も対話的なフロントエンドである UI を開きます）。worktree を作るときは
従来どおり素の名前を渡します（`wt <name>` ≡ `wt create <name>`）。

![wt ui のデモ（ツリーからサーバーを選択 → ログを追従 → switch / stop → 作成・削除）](assets/tui-demo.gif)

## レイアウト

```
╭─ WORKTREES ─────────────╮╭─ ログ: api/backend @ feat-login ──────────╮
│▌▾ feat-login (ログイン)  ││12:00:01 GET /healthz 200                  │
│   ● api/backend :4242   ││12:00:02 listening on :4242                │
│   ○ api/web             ││...                                        │
│ ▾ feat-search           ││                                           │
│   ✗ api/backend         ││                                           │
│   ○ api/web             ││                                           │
│ ▸ feat-old  ○2          ││                                           │
╰─────────────────────────╯╰───────────────────────────────────────────╯
j/k 選択 · J/K wt 移動 · Space 折りたたみ · …
```

- **左ペイン（ツリー）** — worktree を見出し（展開中は `▾`、折りたたみ中は `▸`）に、
  その配下へメンバーリポジトリのサーバーをぶら下げます。別名は worktree 名の後ろに
  `(…)`、壊れたチェックアウトは `(!)` を付けます。サーバーノードの状態はマークで
  示します:

  | マーク | 状態 |
  | --- | --- |
  | `●`（緑） | 稼働中（`:pid` を併記） |
  | `✗`（赤） | クラッシュ（記録はあるがプロセスが消滅） |
  | `○`（faint） | 停止 |

- **worktree の折りたたみ** — 複数リポジトリ構成ではツリーが「worktree 数 × 全
  リポジトリのサーバー数」で縦に伸び、停止中の worktree の行はほぼ冗長になります。
  折りたたむと配下のサーバーノードを隠し、見出しの右へ状態を集約したマーク
  （例: `▸ feat-old  ○2`、`▸ feat-x  ●2 ✗1`。0 件の状態は表示しません）だけを
  残します。既定では、**稼働中またはクラッシュのサーバーを 1 つでも持つ worktree は
  展開、すべて停止なら折りたたみ**ます。`Space` で明示的にトグルすると、以後その
  worktree は既定ルールより明示指定が優先されます（折りたたんでもログの表示対象は
  維持されます）。

- **右ペイン（ログ）** — カーソルが乗っているサーバー（サーバーノード上に無い間は
  直近の対象）のログを増分読み（tail -f 相当の Go 実装）で表示します。末尾 64KB から
  読み始めるため巨大なログでも即座に開きます。

`Tab`（または `←`/`→`・`h`/`l`）でペインのフォーカスを切り替えます。フォーカス側は
ペインのボーダーと見出しがアクセント色になり、ツリーの選択行は行頭の `▌` で示します。
終了は `q` です。ログ見出しには状態のバッジが付きます（追従を解除している間の
`[追従停止]`、`[前世代]`・`[/検索語]`・`[読取失敗]`。既定の追従中はバッジなし）。

## ツリーのキー（左ペイン）

| キー | 動作 | 対応する CLI |
| --- | --- | --- |
| `j` / `k` | カーソル移動（サーバーノードに乗るとログ対象が切り替わる） | — |
| `J` / `K` | 次 / 前の worktree 見出しへ移動（間のサーバーノードを飛ばす。端では停止） | — |
| `Space` | カーソル位置の worktree（サーバーノード上なら親）の折りたたみをトグル | — |
| `Enter` / `s` | 選択 worktree へサーバーを切り替え | `wt server switch <name>` |
| `r` | 既に稼働中でも再起動して切り替え | `wt server switch --restart <name>` |
| `x` | 選択 worktree のサーバーを停止 | `wt server stop <name>` |
| `n` | worktree を作成（名前と作成先リポジトリを 1 枚のフォームで入力） | `wt create <name>` |
| `D` | 選択 worktree を削除（確認フォームで実行／中止） | `wt remove <name>` |
| `a` | 選択 worktree の別名を設定／削除 | `wt alias set` / `wt alias remove` |
| `!` | 自己診断を実行（結果は右ペインに表示） | `wt doctor` |
| `R` | 一覧を再取得 | `wt list` |
| `Tab` | ログペインへフォーカス移動 | — |

## ログのキー（右ペイン）

| キー | 動作 |
| --- | --- |
| `f` | 末尾追従（tail -f 相当）のオン／オフ。上へスクロールすると自動で解除され、解除中はログ見出しに `[追従停止]` が付く |
| `/` | キーワードで絞り込み（部分一致・大文字小文字を無視。1 打鍵ごとに反映） |
| `p` | 前世代のログ（サーバー起動時にローテートされた `.prev`）との切り替え |
| `w` | 長い行の折り返しのオン／オフ |
| `j` / `k` / `d` / `u` / `g` / `G` | スクロール（1 行 / 半ページ / 先頭 / 末尾） |
| `Esc` | フィルタ解除 |
| `Tab` | ツリーペインへフォーカス移動 |

`error` / `warn` などの語を含む行は自動で色付けされます（ヒューリスティック）。

## プロンプト・結果ペインのキー

作成・別名・削除の確認は [huh](https://github.com/charmbracelet/huh) のフォームで
入力します。フィルタ（`/`）だけはライブ反映のインライン入力のため従来どおりです。

| 文脈 | キー | 動作 |
| --- | --- | --- |
| 作成フォーム（`n`） | `Tab` / `Enter` | 次のフィールドへ（最終フィールドの `Enter` は確定して作成を実行） |
| 作成フォーム（`n`） | `Shift+Tab` | 前のフィールドへ戻る |
| 作成フォーム（`n`） | `↑` / `↓` | リポジトリ選択内のカーソル移動 |
| 作成フォーム（`n`） | `space` / `x` | カーソル位置のリポジトリの選択を切替 |
| 作成フォーム（`n`） | `ctrl+a` | リポジトリを全選択／全解除 |
| 作成フォーム（`n`） | `/` | リポジトリを絞り込み（`Esc` で確定、もう一度 `Esc` で解除） |
| 別名フォーム（`a`） | `Enter` / `Esc` | 確定（空で削除） / 中止 |
| 削除の確認（`D`） | `←`/`→`・`Enter` / `Esc` | 「削除する／やめる」を選び確定 / 中止 |
| フォーム共通 | `Esc` | 中止（何もせずに閉じる） |
| doctor 結果 | `F`・`j`/`k`・`d`/`u`・`g`/`G`・`Esc` | `--fix` で再実行 / スクロール（1 行 / 半ページ / 先頭・末尾） / 閉じる |

## TUI の各操作と CLI コマンドの対応

TUI は既存の CLI を置き換えるものではなく、**同じワークフローを呼ぶ追加のフロント
エンド**です。すべての操作は従来どおり CLI からも実行できます。

| TUI の操作 | 対応する CLI コマンド |
| --- | --- |
| `n`（作成） | `wt create <name>` |
| `D`（削除） | `wt remove <name>` |
| `a`（別名設定／削除） | `wt alias set <name> <label>` / `wt alias remove <name>` |
| `!`（診断・修復） | `wt doctor` / `wt doctor --fix` |
| `Enter` / `s`（切り替え） | `wt server switch <name>` |
| `r`（再起動して切り替え） | `wt server switch --restart <name>` |
| `x`（停止） | `wt server stop <name>` |
| ログ閲覧 | `wt server logs [-f] <name>` |

TUI からの削除は常に非 `--force` です。dirty なチェックアウトを git が拒否した場合は
`wt remove --force <name>` を CLI から実行してください（LLM も TUI も、dirty の強制
削除は明示コマンド経由に限ります）。

## MCP・他プロセスとの兼ね合い

`ui` は MCP サーバーや別のシェルの CLI と**並行して動かす**前提で設計されています。

- **外部の変更に自動追従します。** 選択中サーバーのログパス・サーバー状態・worktree
  一覧は定期的に再解決されるため、LLM エージェントが MCP の `server_switch` で worktree を
  切り替えると、ログ表示は数秒以内に新しい対象へ自動で切り替わり、ツリーのマークや
  一覧も同様に追従します。
- **操作は同じロックを通ります。** TUI からの操作は CLI / MCP と同じ repo 操作ロックを
  通るため、エージェントの操作と衝突してもリポジトリ単位で直列化されます。TUI だけの
  特別な経路はありません。
- **MCP から TUI は起動できません。** `ui` は `server logs -f` と同じ CLI 専用の
  実行モードであり、MCP のツール語彙に存在しません（stdio を JSON-RPC が占有する
  MCP の下で全画面 UI が端末を奪う事故は型レベルで起こりません）。
- **設定はティックごとに再読み込みされます。** MCP サーバーと同じく、設定ファイルの
  編集は TUI の再起動なしで反映されます（編集途中で不正な間は直近の正常な設定で
  動き続けます）。

なお、TUI が端末を専有している間、ライフサイクルコマンド（`setup` / `on_switch`
など）の出力は表示されません（画面破壊を防ぐため破棄されます）。失敗時はエラーと
起動失敗のログ末尾がイベント・メッセージとして報告されます。詳細な出力が必要な
場合は CLI の `wt server switch` を使ってください。
